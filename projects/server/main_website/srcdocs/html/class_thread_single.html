Detailed Description

Base class to be inherited for spawning a single workerBee() pure virtual.

For multiple threads, use ThreadMultiple instead.

Details: Simply inherit this class and create your own workerBee() function in the parent. Then, use startThread() to start the new thread which will call your workerBee from the newly created thread. When it is time to shutdown, simply call stopThread() and when it returns, you're done.

Note:
It is important to note that your implementation of workerBee() must be looking for the isStopRequested flag to signal it to return() - otherwise the thread class will not end nicely and will crash upon a change of scope and stack.
Also, there is a built in single mutex created at instantiation time. Parent may pthread_mutex_lock(&mutex) and then can pthread_mutex_unlock(&mutex) or even using signal/wait actions as well.

void ThreadSingle::startThread ( ) [inline]

Parent calls startThread() when they are ready to launch/fork/spawn the new thread.

Once this function returns, the pthread_create() within the function will cause workerBee() to be called. The parent must implement workerBee() for its work item(s) and/or calls outwards.

NOTE: isStopRequested is a variable which must be monitored by workerBee() to know when it is being told to stop (return).

void ThreadSingle::stopThread ( ) [inline]

Parent calls stopThread() when they are ready to close/stop the thread.

Once this function returns, the thread is no longer active. It has been destroyed. The parent, in its implementation of workerBee() must monitor 'isStopRequested' or it will never know when to quit itself via a return() call.

virtual int ThreadSingle::workerBee ( ) [protected, pure virtual]

Parent implements this pure virtual in its class.

This is where the new thread has its entry point. All work is done here (or in subsequent calls) to other functions. The biggest items to note are 'be kind - no tight loops for long periods' and also to monitor and adhere quickly to 'isStopRequested' flag by calling return().

NOTE: isStopRequested is a variable which must be monitored by workerBee() to know when it is being told to stop (return).

Implemented in PluginNotificationsThread, and PluginMainThread.
