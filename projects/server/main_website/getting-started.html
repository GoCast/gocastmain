Getting Started

There are 2 main requirements tools-wise for setup of WebRTC, FireBreath, and our additions. The first is 'git'. If you type 'which git' a the command line and get a response with a path to the tool, you're likely just fine. If not, please go to http://www.git-scm.org and download the sources or a binary package and install. In most cases for linux, you can 'sudo apt-get install git' to accomplish this. The second tool is 'cmake', if you do not have this tool it can be obtained from cmake.org.

The next requirement is the Chromium project build tools. The tool 'gclient' is the key item. If you do not have gclient, you may go to the page which describes installation of the tool. The prep* scripts do indeed self-check that git and gclient are available and will exit if they are not present.

After cloning the git repository, there are a number of items which need to be prepared for initial use. Webrtc can be a difficult build procedure and so we've tried to automate this as much as possible.

./prep_webrtc.sh -- this is the main script which will kick off all further processing to get an initial set of compiled binaries in both Debug and Release modes. It will (when successful) get a 'blessed' version of WebRTC, patch it as necessary, and then build the WebRTC libraries necessary to support cmdline_audio_peer, peerconnection_server, and the audio call plugin. It will then create a monolithic library named 'libwebrtc_Release.a' and 'libwebrtc_Debug.a' in the output directories. These libraries can significanly ease the use of WebRTC by developers wishing to use the 'guts' programmatically rather than from a browser. At the end of prep_webrtc.sh, you'll have the monolithic static libs along with the binaries for the above mentioned items. Lastly, this set of scripts calls rebuild_plugin.sh which will build the FireBreath based plugin for the audio call plugin project. See Give it a try for step by step instructions to validate your binaries are functional.
